Lesson 2: Creating a Structured Document with HTML

Developer Tools

Everything in a web page is inside a box!
Devs first envision the page as a group of boxes, some nested inside others.

The Tree structure of HTML

This means that these "boxes" can be inside others, to keep the page organized and grouped by relationships. We use the div tag to create this boxes.

Indentation in HTML

The indentation can be seen on the left of the html code. This shows the nesting happening among these boxes.

Text Editors

There are many ways to write code, even TextEdit on Mac OS or notepad on Windows, but there are some editors that aid programmers during the writing code process, but autocompleting your html tags and closing them. Putting colors to differnt types of text, to easily identify them.

The best editors out there are Sublime Text, TextMate or online tools like Scratchpad or Codepen.

Lesson 3: Adding CSS to your HTML structure

Avoiding Repetition

When giving all the style to the page. We can do it in a very efficient way, by avoiding repeating the code for similar elements in the page.

This is accomplished by organizing our elements in the html structure, and grouping similar elements under the same type (class), so we can style them all at the same time, in an identical way.

CSS: Cascading Style Sheet

CSS defines the style for the html document, and this is done by adding this style to specific classes that have been defined in the html document.
For example:
All headers (h{1..6}) in a document can have a different styles if they were assigned different class names.

Also, CSS class attributes are inherited to the elements contained in it, and only overriden if there is a class definition more specific to that element.
